# Checkbox

Default checkboxes improved with the help of `.form-check`, a single class that improves the layout and behavior of their HTML elements. Checkboxes are for selecting one or several options in a list, while radios are for selecting one option from many.

Checkboxes are built to support HTML-based form validation and provide concise, accessible labels. As such, our `<input>` and `<label>` are sibling elements as opposed to an `<input>` within a `<label>`. This is slightly more verbose as you must specify id and for attributes to relate the `<input>` and `<label>`.

<ComponentPreview>
	<Form>
		<FormGroup>
			<CustomInput
				type="checkbox"
				id="exampleCustomCheckbox1"
				label="Check this out"
			/>
		</FormGroup>
	</Form>
</ComponentPreview>

```  html
    <div class="form-check">
        <input class="form-check-input" type="checkbox" name="exampleCustomCheckbox" id="customCheckbox1" value="option1" checked>
        <label class="form-check-label" for="customCheckbox1">
            Check this out
        </label>
    </div>
```

### Disabled Checkbox

Disabled checkboxes and radios are supported. The disabled attribute will apply a lighter color to help indicate the input’s state.

<ComponentPreview>
	<Form>
		<FormGroup>
			<CustomInput
				type="checkbox"
				id="exampleCustomCheckbox2"
				label="But don't check this"
				disabled
			/>
		</FormGroup>
	</Form>
</ComponentPreview>

```  html
    <div class="form-check">
        <input class="form-check-input" type="checkbox" name="exampleCustomCheckbox1" id="customCheckbox3" value="option3" disabled>
        <label class="form-check-label" for="customCheckbox3">
            But don't check this
        </label>
    </div>
```

### Default (Stacked)

By default, any number of checkboxes and radios that are immediate siblings will be vertically stacked and appropriately spaced with `.form-check`.

<ComponentPreview>
	<Form>
		<FormGroup>
			<CustomInput
				type="checkbox"
				id="exampleCustomCheckbox3"
				label="Default checkbox"
			/>
		</FormGroup>
		<FormGroup>
			<CustomInput
				type="checkbox"
				id="exampleCustomCheckbox4"
				label="Default checkbox"
			/>
		</FormGroup>
	</Form>
</ComponentPreview>

```  html
    <div class="form-check">
        <input class="form-check-input" type="checkbox" value="" id="defaultCheck1">
        <label class="form-check-label" for="defaultCheck1">
            Default checkbox
        </label>
    </div>
    <div class="form-check">
        <input class="form-check-input" type="checkbox" value="" id="defaultCheck2">
        <label class="form-check-label" for="defaultCheck2">
            Disabled checkbox
        </label>
    </div>
```

### Inline

Group checkboxes or radios on the same horizontal row by adding `.form-check-inline` to any `.form-check`.

<ComponentPreview>
	<Form>
		<FormGroup check inline>
			<CustomInput
				type="checkbox"
				id="exampleCustomCheckbox5"
				label="Inline checkbox 1"
			/>
		</FormGroup>
		<FormGroup check inline>
			<CustomInput
				type="checkbox"
				id="exampleCustomCheckbox6"
				label="Inline checkbox 2"
			/>
		</FormGroup>
		<FormGroup check inline>
			<CustomInput
				type="checkbox"
				id="exampleCustomCheckbox7"
				label="Inline checkbox 3"
			/>
		</FormGroup>
	</Form>
</ComponentPreview>

```  html
    <div class="form-check form-check-inline">
        <input class="form-check-input" type="checkbox" id="inlineCheckbox1" value="option1">

    </div>
    <div class="form-check form-check-inline">
        <input class="form-check-input" type="checkbox" id="inlineCheckbox2" value="option2">

```

### Assistive Technology

Add `.position-static` to inputs within `.form-check` that don’t have any label text. Remember to still provide some form of label for assistive technologies (for instance, using `aria-label`).

    <div class="form-check">
        <input class="form-check-input position-static" type="checkbox" id="blankCheckbox" value="option1" aria-label="...">
    </div>
