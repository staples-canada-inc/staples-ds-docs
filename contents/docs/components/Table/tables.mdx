# Tables

Due to the widespread use of tables across third-party widgets like calendars and date pickers, we've designed our tables to be opt-in. Just add the base class `.table` to any `<Table>`, then extend with custom styles or our various included modifier classes.

### Assistive Technology

Using colour to add meaning only provides a visual indication, which will not be conveyed to users of assistive technologies – such as screen readers. Ensure that information denoted by the colour is either obvious from the content itself (e.g. the visible text), or is included through alternative means, such as additional text hidden with the `.sr-only` class.

## Responsive Tables

Responsive tables allow tables to be scrolled horizontally with ease. Make any table responsive across all viewports by wrapping a `.table` with `.table-responsive`.

Responsive tables make use of `overflow-y: hidden`, which clips off any content that goes beyond the bottom or top edges of the table. In particular, this can clip off dropdown menus and other third-party widgets.

### Breakpoint Specific

Use `.table-responsive{-sm|-md|-lg|-xl}` as needed to create responsive tables up to a particular breakpoint. From that breakpoint and up, the table will behave normally and not scroll horizontally.

**These tables may appear broken until their responsive styles apply at specific viewport widths.**

#### Small

```  html
        <div class="table-responsive-sm">
            <table class="table">
                ...
            </table>
        </div>
```

#### Medium

```  html
        <div class="table-responsive-md">
            <table class="table">
                ...
            </table>
        </div>
```

#### Large

```  html
        <div class="table-responsive-lg">
            <table class="table">
                ...
            </table>
        </div>
```

#### Extra-Large

```  html
        <div class="table-responsive-xl">
            <table class="table">
                ...
            </table>
        </div>
```

**Note:** Since browsers do not currently support `range context queries`, we work around the limitations of `min- and max- prefixes` and viewports with fractional widths (which can occur under certain conditions on high-dpi devices, for instance) by using values with higher precision for these comparisons.
