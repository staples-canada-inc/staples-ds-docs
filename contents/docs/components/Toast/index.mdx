# Toasts

Toasts are lightweight notifications designed to mimic the push notifications that have been popularized by mobile and desktop operating systems. They’re built with flexbox, so they’re easy to align and position.

## Basic Toasts

To encourage extensible and predictable toasts, we recommend a header and body. Toast headers use `display: flex`, allowing easy alignment of content thanks to our margin and flexbox utilities.

Toasts are as flexible as you need and have very little required markup. At minimum, a single element is required to contain your "toasted" content with a dismiss button.

## Translucent Toasts

Toasts are slightly translucent, and can blur the elements they appear over in browsers that support the `backdrop-filter` CSS property.

```html
<div class="toast" role="alert" aria-live="assertive" aria-atomic="true">
  <div class="toast-header">
    <img src="..." class="rounded mr-2" alt="..." />
    <strong class="mr-auto">Staples</strong>
    <small class="text-muted">11 mins ago</small>
    <button
      type="button"
      class="ml-2 mb-1 close"
      data-dismiss="toast"
      aria-label="Close"
```

## Position

You can customize the placement of toasts as needed with css. The top right of an interface is commonly used for notifications, as is the top middle. However, if you intend on displaying toasts one at a time, you can specify the position in the `.toast` class itself.

## Autohide

As the content you're displaying changes, be sure to update the [`delay` timeout](#options) to ensure people have enough time to read the toast.

```html
  role="alert"
  aria-live="polite"
  aria-atomic="true"
  data-delay="10000"

  <div role="alert" aria-live="assertive" aria-atomic="true">...</div>
```

## Assistive Technology

To help screen readers and similar assistive technologies treat toasts as they're intended to be, you should wrap your toasts in an `aria-live` region. Changes to live regions (such as injecting/updating a toast component) are automatically announced by screen readers without having to move the user's focus or direct their attention elsewhere. Additionally, include `aria-atomic="true"` to ensure that the entire toast is always announced as a single unit, rather than announcing just the changes.

The live region needs to be present in the markup _before_ the toast is generated or updated. If you dynamically generate both at the same time and inject them into the page, they will not be announced by assistive technologies.

Do not use toasts to convey essential information. If the notification's content is important to the process, such as a list of errors in a form, then it is best to use [alerts](#) instead.

You also need to adapt the `role` and `aria-live` level depending on the content. If it's an important message like an error, use `role="alert" aria-live="assertive"`, otherwise use `role="status" aria-live="polite"` attributes.

## Usage

Unlike a modal, a toast is an unobtrusive element used to display brief, auto-expiring snippets of information to a user. They are often used as notifications for messages, actions (adding cart items) and/or updates, to name a few.

### Overview

Toasts will automatically hide if you do not specify `autohide: false`. A close button should also be added to allow users to dismiss the toast.

Since toasts are opt-in for performance reasons, **you must initialize them yourself**. You are also required to position them as well.

### JavaScript Behavior

Initialize toasts via JavaScript:

```javascript
$('.toast').toast(option)
```

### Options

Options can be passed via data attributes or JavaScript. For data attributes, append the option name to `data-`, as in `data-animation=""`.

#### \$().toast(options)

Attaches a toast handler to an element collection.

#### .toast('show')

Reveals an element’s toast. Returns to the caller before the toast has actually been shown (i.e. before the `shown.bs.toast` event occurs). You have to manually call this method, instead `your toast won’t show.

```javascript
$('#element').toast('show')
```

#### .toast('hide')

Hides an element’s toast. Returns to the caller before the toast has actually been hidden (i.e. before the `hidden.bs.toast` event occurs). You have to manually call this method if you made `autohide` to `false`.

```javascript
$('#element').toast('hide')
```

#### .toast('dispose')

Hides an element’s toast. Your toast will remain on the DOM but won’t show anymore.

```javascript
$('#element').toast('dispose')
```
