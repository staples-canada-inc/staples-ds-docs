# Popovers

The Popover plugin is similar to tooltips in that a pop-up box appears upon click of an element. The difference is that a popover can contain much more content.

## Buttons with Popovers

You can use buttons that will activate a popover on click, and dismiss the popover after another click.

```html
<button
  type="button"
  class="btn btn-lg btn-danger"
  data-toggle="popover"
  data-content="And here's some amazing content. It's very engaging. Right?"
>
  Click to toggle popover
</button>
```

You can also **add headers** in popovers.

```html
<button
  type="button"
  class="btn btn-lg btn-danger"
  data-toggle="popover"
  title="Popover title"
  data-content="And here's some amazing content. It's very engaging. Right?"
>
  Click to toggle popover
</button>
```

## Text with Popovers

Not only do popvers work with buttons, but they can also work with links. Again, a click on the text will activate a popover upon click and will dismiss the popover after another click.

```html
<a
  href="#"
  data-toggle="popover"
  title="Popover Header"
  data-content="Some content inside the popover"
  >Toggle popover</a
>
```

## Popover Direction

Popovers can open in one of four directions (top, right, bottom, or left).

>
  Popover on top

## Dismissable Popovers

Instead of clicking the same hit area that was used to activate the popover, in order to dismiss it, you can use the `focus` trigger. This allows the user to click on any other element to dismiss the popover.

### Dismiss-on-next-click

For proper cross-browser and cross-platform behavior, you must use the `<a>` tag and include a `tabindex` attribute. Do not use the `<button>` tag.

```html
<a
  tabindex="0"
  class="btn btn-lg btn-danger"
  role="button"
  data-toggle="popover"
  data-trigger="focus"
  title="Dismissible popover"
  data-content="And here's some amazing content. It's very engaging. Right?"
  >Dismissible popover</a
>
```

### Disabled elements

Elements with the `disabled` attribute aren't interactive, meaning users cannot hover or click them to trigger a popover (or tooltip). As a workaround, you'll want to trigger the popover from a wrapper `<div>` or `<span>` and override the `pointer-events` on the disabled element.

For disabled popover triggers, you may also consider `data-trigger="hover"` so that the popover appears to provide immediate visual feedback to the user. This works in the case that a user is not expected to click on a disabled element.

```html
<span
  class="d-inline-block"
  data-toggle="popover"
  data-content="Disabled popover"
>
  <button
    class="btn btn-primary"
    style="pointer-events: none;"
    type="button"
    disabled
  >
    Disabled button
  </button>
</span>
```

## Hoverable Popovers

If you want the popover to display only on hover, use the `data-trigger` attribute with a value of `"hover"`:

```html
<a
  href="#"
  title="Header"
  data-toggle="popover"
  data-trigger="hover"
  data-content="Some content"
  >Hover over me</a
>
```

## Usage

To create a popover, add the `data-toggle="popover"` attribute to an element.

Use the `title` attribute to specify the header text of the popover, and use the `data-content` attribute to specify the text that should be displayed inside the popover.

Enable popovers via JavaScript:

```javascript
$('#example').popover(options)
```

### Enable Popovers Everywhere

One way to initialize all popovers on a page would be to select them by their `data-toggle` attribute:

```javascript
$('[data-toggle="popover"]').popover()
```

### Using a `container`

When you have some styles on a parent element that interfere with a popover, you'll want to specify a custom `container` so that the popover's HTML appears within that element instead.

```javascript
$('.example-popover').popover({
  container: 'body',
})
```

## Assistive Technology

To allow keyboard users to activate popovers, you should only add them to HTML elements that are traditionally keyboard-focusable and interactive (such as links or form controls). Although arbitrary HTML elements (such as `<span>`) can be made focusable by adding the `tabindex="0"` attribute, this will add potentially annoying and confusing tab stops on non-interactive elements for keyboard users. Not to mention, most assistive technologies do not announce the popover content in this situation. It is also good practice to not rely solely on `hover` as a trigger for your popovers, since this will make it impossible for keyboard users to trigger.

While you can insert rich, structured HTML in popovers with the `html` option, it is strongly advised to avoid adding a lot of content. The way popovers currently work is that, once displayed, their content is tied to the trigger element with the `aria-describedby` attribute. As a result, the entirety of the popover's content will be announced to assistive technology users as one long, uninterrupted stream.

#### Data attributes for individual popovers

Options for individual popovers can alternatively be specified through the use of data attributes, as explained above.

#### \$().popover(options)

Initializes popovers for an element collection.

#### .popover('show')

Reveals an element’s popover. Returns to the caller before the popover has actually been shown (i.e. before the `shown.bs.popover` event occurs). This is considered a “manual” triggering of the popover. Popovers whose both title and content are zero-length are never displayed.

```javascript
$('#element').popover('show')
```

#### .popover('hide')

Hides an element’s popover. Returns to the caller before the popover has actually been hidden (i.e. before the `hidden.bs.popover` event occurs). This is considered a “manual” triggering of the popover.

```javascript
$('#element').popover('hide')
```

#### .popover('toggle')

Toggles an element’s popover. Returns to the caller before the popover has actually been shown or hidden (i.e. before the `shown.bs.popover` or `hidden.bs.popover` event occurs). This is considered a “manual” triggering of the popover.

```javascript
$('#element').popover('toggle')
```

#### .popover('dispose')

Hides and destroys an element’s popover. Popovers that use delegation (which are created using [the selector option](https://getbootstrap.com/docs/4.3/components/popovers/#options) cannot be individually destroyed on descendant trigger elements.

```javascript
$('#element').popover('dispose')
```

#### .popover('enable')

Gives an element’s popover the ability to be shown. **Popovers are enabled by default**.

```javascript
$('#element').popover('enable')
```

#### .popover('disable')

Removes the ability for an element’s popover to be shown. The popover will only be able to be shown if it is re-enabled.

```javascript
$('#element').popover('disable')
```

#### .popover('toggleEnabled')

Toggles the ability for an element’s popover to be shown or hidden.

```javascript
$('#element').popover('toggleEnabled')
```

#### .popover('update')

Updates the position of an element’s popover.

```javascript
$('#element').popover('update')
```
